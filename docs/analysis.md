# تحليل مشروع تطبيق عقاري (DEALAK)

**الجهة:** وزارة التعليم العالي والبحث العلمي - جامعة حماة - المعهد التقاني للحاسوب  
**القسم:** قسم هندسة البرمجيات  
**المادة:** تحليل نظم المعلومات  
**الفصل الدراسي:** الأول لعام 2025/2026 م  
**المشرف:** المهندسة فاطمة الشيخ صبح

1. مريم البكور
2. سدرة بظ
3. بيان غبشة
4. كرم الحمد

## 🎯 نظرة عامة على النظام

### الوصف العام
منصة متكاملة تقدم خدمات إيجاد عقارات بشكل أسرع للمواطنين داخل الأراضي السورية من خلال تطبيق هاتف ذكي وموقع إلكتروني متصلين بنفس قاعدة البيانات.

### المكونات الرئيسية
1. **واجهة المستخدم (User Interface):** واجهة تفاعلية بسيطة وسهلة الاستخدام للمستخدمين والمالكين
2. **نظام تحديد الموقع:** يستخدم GPS لتحديد موقع العقار بدقة
3. **نظام المحادثة:** يسمح بالتواصل بين المستخدم ومالك العقار
4. **التكامل مع خدمات خارجية:**
   - خرائط Google للعرض الجغرافي
   - نظام إرسال إشعارات
   - بوابات دفع إلكتروني للحجز المدفوع

## 👥 المستخدمون وأدوارهم

### 1. المستخدم (الزبون)
**الوصف:** مواطن يبحث عن عقار للبيع أو الإيجار بمواصفات محددة

**المهام الرئيسية:**
- البحث عن العقار
- إرسال طلب عقار
- تحديد نوع الطلب (بيع/إيجار)
- تحديد مواصفات العقار المطلوبة (مساحة، موقع، سعر)

### 2. المستخدم (مالك العقار)
**الوصف:** مواطن يعرض العقار الذي يريد تأجيره أو بيعه

**المهام الرئيسية:**
- استلام الطلبات العقارية
- التواصل مع الزبون
- الاطلاع على المواصفات المطلوبة من قبل الزبون

### 3. مدير النظام
**الوصف:** شخص مسؤول عن إدارة المنصة ومراقبة الطلبات والإعلانات

**المهام الرئيسية:**
- مراجعة الحسابات
- إدارة البيانات العامة
- متابعة الطلبات والشكاوى
- الرد على الاستفسارات

## 🗄️ تحليل ERD (مخطط قاعدة البيانات)

### الكيانات الرئيسية

#### 1. Users (المستخدمون)
- **الحقول:**
  - `id`: المعرف
  - `name`: الاسم
  - `charachtar`: الخصائص
  - `phone`: الهاتف
  - `email`: البريد الإلكتروني

#### 2. Property (العقارات)
- **الحقول:**
  - `id`: المعرف
  - `desc`: الوصف
  - `space`: المساحة
  - `featurse`: المميزات
  - `situation`: الحالة
  - `type`: النوع
  - `address`: العنوان
  - `tax`: الضريبة
  - `price`: السعر

#### 3. Picture (الصور)
- **الحقول:**
  - `id`: المعرف
  - `desc`: الوصف
  - `link`: الرابط
  - `publication_date`: تاريخ النشر

#### 4. Notice (الإشعارات)
- **الحقول:**
  - `id`: المعرف
  - `type`: النوع
  - `adv_id`: معرف الإعلان

#### 5. Deal (الصفقات)
- **الحقول:**
  - `id`: المعرف
  - `price`: السعر
  - `type`: النوع
  - `deal_desc`: وصف الصفقة
  - `release_date`: تاريخ الإصدار
  - `currency`: العملة

#### 6. Payment (الدفعات)
- **الحقول:**
  - `id`: المعرف
  - `value`: القيمة
  - `type`: النوع
  - `details`: التفاصيل

#### 7. Discount (الخصومات)
- **الحقول:**
  - `id`: المعرف
  - `value`: القيمة
  - `amount_after_discount`: المبلغ بعد الخصم
  - `amount_before_discount`: المبلغ قبل الخصم

#### 8. Request (الطلبات)
- **الحقول:**
  - `id`: المعرف
  - `type`: النوع
  - `property_type`: نوع العقار
  - `request`: الطلب
  - `address`: العنوان
  - `space`: المساحة
  - `maximum_budget`: الحد الأقصى للميزانية
  - `price`: السعر

#### 9. Deal Information (معلومات الصفقة)
- **الحقول:**
  - `id`: المعرف
  - `customer_name`: اسم الزبون
  - `seller_name`: اسم البائع
  - `customer_number`: رقم الزبون
  - `seller_number`: رقم البائع
  - `signature`: التوقيع

### العلاقات
- **Users ↔ Property:** علاقة واحد-إلى-متعدد (1:n)
- **Property ↔ Picture:** علاقة واحد-إلى-متعدد (1:n)
- **Property ↔ Notice:** علاقة واحد-إلى-متعدد (1:n)
- **Deal ↔ Payment:** علاقة واحد-إلى-متعدد (1:n)
- **Deal ↔ Discount:** علاقة واحد-إلى-متعدد (1:n)
- **Deal ↔ Request:** علاقة واحد-إلى-متعدد (1:n)
- **Deal ↔ Deal Information:** علاقة واحد-إلى-واحد (1:1)

## 📊 تحليل Use Case Diagram

### المشاركون (Actors)
1. **Customers (الزبائن)**
2. **Seller (البائع/مالك العقار)**
3. **System Administrator (مدير النظام)**

### حالات الاستخدام الرئيسية

#### للزبائن (Customers):
1. **Browsing (تصفح):** تصفح العقارات المتاحة
2. **Request a property (طلب عقار):** إرسال طلب لعقار معين
3. **Receives a notification (استلام إشعار):** استلام إشعارات
4. **Sign a deal (توقيع صفقة):** توقيع اتفاقية
5. **Pay the amount (دفع المبلغ):** دفع قيمة العقار
6. **Create an account (إنشاء حساب):** تسجيل حساب جديد
7. **Communication (التواصل):** التواصل مع البائع

#### للبائع (Seller):
1. **Advertising a property (إعلان عقار):** نشر عقار للبيع/للإيجار
2. **Add a picture (إضافة صور):** إضافة صور للعقار
3. **Receives requests (استلام طلبات):** استلام طلبات من الزبائن
4. **Acceptance/Reject (القبول/الرفض):** قبول أو رفض الطلبات
5. **Send a notification (إرسال إشعار):** إرسال إشعارات للزبائن
6. **Create a deal (إنشاء صفقة):** إنشاء صفقة جديدة
7. **Payment continues (استمرار الدفع):** متابعة عمليات الدفع

#### لمدير النظام (System Administrator):
1. **User management (إدارة المستخدمين):** إدارة حسابات المستخدمين
2. **Manages requests (إدارة الطلبات):** مراقبة وإدارة الطلبات
3. **Manages advertisements (إدارة الإعلانات):** إدارة الإعلانات
4. **Manages notifications (إدارة الإشعارات):** إدارة نظام الإشعارات
5. **Track deals (تتبع الصفقات):** متابعة الصفقات
6. **Track payments (تتبع الدفعات):** متابعة عمليات الدفع
7. **Add discount (إضافة خصم):** إضافة خصومات
8. **Manage settings (إدارة الإعدادات):** إدارة إعدادات النظام

## ⚙️ القيود (Constraints)

### 1. القيود التقنية
- **الاعتماد على الاتصال بالإنترنت:** يتطلب النظام اتصالاً دائماً بالإنترنت لأداء معظم المهام مثل تحديد الموقع وإجراء الحجز
- **توافر خدمة GPS:** يعتمد النظام على خدمة تحديد الموقع للعقار بشكل دقيق
- **اللغة المدعومة:** النظام مصمم في إصداره الأولي لدعم اللغة العربية والإنكليزية

### 2. قيود المنصات
- يعمل النظام على منصات iOS و Android في الإصدار الأول

### 3. قيود البيانات والأمان
- يتم حفظ بيانات المستخدمين بشكل آمن بعد موافقتهم
- في الإصدار الأول، يدعم النظام العملات المحلية فقط

### 4. قيود الأداء
- يجب أن يستجيب النظام في غضون ثانيتين لطلبات المستخدم

## 🔌 واجهات النظام (System Interfaces)

### 1. واجهة المستخدم (User Interface)
**الوصف:** واجهة تفاعلية تعمل على تطبيق الهاتف والموقع الإلكتروني

**الخصائص:**
- تصميم جذاب وألوان مريحة للعين
- متجاوب (responsive) مع جميع الأجهزة (موبايل، تابلت، لابتوب)
- سهولة التنقل بين الصفحات (صفحة البحث، صفحة الطلب، الصفحة الرئيسية)
- إمكانية البحث والتصفية بسهولة
- إمكانية إضافة العقارات والتعليقات

### 2. واجهة API (API Interface)
**الوصف:** واجهة للاتصال بين النظام الخلفي والواجهات الأمامية

**الخصائص:**
- إدارة البيانات والمجالات
- توثيق الطلبات والمهام
- استخدام JWT و OAuth للمصادقة
- توحيد Endpoints لجميع المستخدمين

### 3. واجهة قاعدة البيانات (Database Interface)
**الوصف:** واجهة للاتصال بقاعدة البيانات

**الخصائص:**
- استخدام قاعدة بيانات علائقية (PostgreSQL أو MySQL)
- العمليات الأساسية: CRUD (Create, Read, Update, Delete)
- البحث المتقدم عن العقارات على خرائط Google
- حفظ المحادثات
- استخدام Foreign Keys لربط الجداول

## 📱 حالات الاستخدام التفصيلية (Use Cases)

### 1. تصفح العقارات
**الوصف:** يقوم المستخدم بالدخول إلى الصفحة الرئيسية للنظام

**الخطوات:**
- عرض العقارات المتاحة في النظام
- يمكن للمستخدم البحث والتصفية حسب:
  - نوع العقار
  - المساحة
  - السعر
  - الموقع

### 2. عرض تفاصيل العقار
**الوصف:** يقوم المستخدم باختيار عقار معين من العقارات المتاحة

**الخطوات:**
- عرض تفاصيل العقار (المساحة، السعر، الصور)
- الانتقال إلى صفحة تفاصيل العقار

### 3. التواصل بين البائع والزبائن
**الوصف:** يتم التواصل بين البائع والزبائن عبر واجهة المحادثة

**الخطوات:**
- إرسال واستقبال الرسائل
- إرسال مرفقات على العقار المطلوب

## 📎 الملاحظات والتوصيات

### نقاط القوة ✅
1. **تصميم شامل:** المشروع يغطي جميع جوانب تطبيق عقاري متكامل
2. **قاعدة بيانات منظمة:** ERD جيد يغطي جميع الكيانات والعلاقات
3. **متطلبات واضحة:** المتطلبات الوظيفية وغير الوظيفية محددة بوضوح
4. **أمان عالي:** التركيز على حماية البيانات والخصوصية
5. **تجربة مستخدم جيدة:** واجهات سهلة الاستخدام ومتجاوبة

### نقاط تحت تحسين ⚠️
1. **اللغة:** بعض النصوص مكتوبة بالعربية المقلوبة (RTL issue)
2. **التفاصيل التقنية:** يمكن إضافة المزيد من التفاصيل حول:
   - التقنيات المستخدمة (Frontend, Backend, Database)
   - استراتيجية الاختبار
   - خطة النشر
3. **الأمان:** يمكن إضافة:
   - سياسة كلمات المرور
   - نظام المصادقة الثنائية (2FA)
   - تشفير البيانات
4. **الأداء:** يمكن إضافة:
   - استراتيجية التخزين المؤقت (Caching)
   - تحسين الأداء للبحث المتقدم
5. **الاختبار:** يمكن إضافة:
   - خطة اختبار شاملة
   - أنواع الاختبارات (Unit, Integration, E2E)

### توصيات للتنفيذ 🚀
1. **استخدام تقنيات حديثة:**
   - Frontend: React/Next.js أو Vue.js
   - Backend: Node.js/Express أو Python/FastAPI
   - Database: PostgreSQL أو MongoDB
   - Mobile: React Native أو Flutter

2. **البنية المعمارية:**
   - استخدام Microservices Architecture
   - تطبيق RESTful API
   - استخدام Docker للحاويات

3. **الأمان:**
   - تطبيق HTTPS
   - استخدام JWT للمصادقة
   - تشفير البيانات الحساسة

4. **الاختبار:**
   - Unit Testing (Jest, Pytest)
   - Integration Testing
   - E2E Testing (Cypress, Playwright)

5. **النشر:**
   - استخدام CI/CD Pipeline
   - النشر على Cloud (AWS, Google Cloud, Azure)
   - استخدام Kubernetes للتوسع
